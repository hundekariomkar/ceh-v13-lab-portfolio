# 09 – Social Engineering 🎭

## 📌 Lab Overview

This module focuses on social engineering techniques used to manipulate individuals into revealing sensitive information or performing actions that compromise security.

The exercises demonstrated various phishing attack techniques and detection mechanisms within a controlled lab environment.

---

## 🎯 Objectives

- Understand the social engineering attack lifecycle  
- Identify common social engineering techniques  
- Simulate phishing attacks using specialized tools  
- Analyze modern phishing vectors such as QR phishing  
- Perform phishing detection analysis  

---

## 🧠 Social Engineering Attack Lifecycle

1. **Reconnaissance** – Gathering information about the target  
2. **Engagement** – Establishing communication and trust  
3. **Exploitation** – Manipulating the target to achieve attacker objectives  

---

## 📚 Common Types of Social Engineering

- Phishing  
- Spear Phishing  
- Vishing (Voice Phishing)  
- Smishing (SMS Phishing)  
- Pretexting  
- Baiting  
- Tailgating  

---

## 📧 Types of Phishing Attacks

- Email Phishing  
- Spear Phishing  
- Whaling  
- Smishing  
- Vishing  
- Clone Phishing  

---

## 🛠 Tools Used

- Social Engineering Toolkit (SET)  
- CamPhish  
- ShellPhish  
- R3bu5  
- QR Phishing (SET-based simulation)  
- CheckPhish (phishing detection platform)  

---

## 🔍 Methodology

### 🎯 Phishing Attack using SEToolkit

The Social Engineering Toolkit was used to simulate phishing attacks.

Capabilities demonstrated:

- Credential harvesting  
- Fake login page creation  
- Social engineering campaign simulation  

---

### 📱 QR Code Phishing using SEToolkit

QR phishing was simulated to demonstrate how attackers exploit mobile device trust.

Risk demonstrated:

- Redirection to malicious websites  
- Credential capture through mobile phishing  

---

### 📷 Phishing Attack using CamPhish

CamPhish was used to simulate phishing scenarios involving webcam access.

Purpose:

- Demonstrate social engineering exploitation of device permissions  

---

### 🐚 Phishing Attack using ShellPhish

ShellPhish was used to generate phishing pages targeting social media login portals.

Observed risks:

- Credential theft  
- Session compromise  

---

### 🛠 Phishing Attack using R3bu5

R3bu5 was used to simulate additional phishing campaign scenarios.

This helped demonstrate:

- Attack automation  
- Multiple target simulation  

---

### 🔎 Phishing Detection using CheckPhish

CheckPhish platform was used to analyze phishing URLs.

Detection involved:

- URL reputation analysis  
- Domain threat intelligence  
- Phishing pattern identification  

---

## 📊 Findings

- Successfully simulated credential harvesting scenarios  
- Demonstrated effectiveness of QR-based phishing attacks  
- Observed risk of user trust exploitation  
- Identified phishing indicators using detection tools  

---

## ⚠️ Risk Analysis

Social engineering attacks may lead to:

- Credential compromise  
- Unauthorized system access  
- Data breaches  
- Financial fraud  
- Identity theft  

Human factors often represent the weakest security control.

---

## 🛡 Mitigation Recommendations

- Conduct regular security awareness training  
- Implement multi-factor authentication  
- Deploy phishing detection solutions  
- Enforce email security controls  
- Educate users on suspicious communication patterns  

---

## 📝 Lab Outcome

Developed practical understanding of social engineering attack techniques, phishing campaign simulation, and defensive detection strategies within ethical cybersecurity testing environments.
